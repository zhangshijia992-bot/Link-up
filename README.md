# Link-Up AI Matching Desktop Prototype

Link-Up is an AI-powered collaboration and team matching platform. This coded prototype focuses on the key function required for Assignment 2: AI Matching. It also includes supporting pages such as Manual Search, Team Creation, Competitions, Messages, Lifecycle, and Profile so the matching flow feels like a complete platform.

This README is written for beginners. Follow the steps slowly and in order.

## 1. What You Need Before Running

Please install these first:

1. **Visual Studio Code**
2. **Python 3**
3. **Git**
4. A web browser such as Chrome or Edge

To check whether Python is installed, open a terminal and type:

```bash
python --version
```

If it shows a Python version, you can continue.

## 2. Download the Project from GitHub

Open VS Code.

Open the terminal in VS Code:

```text
Terminal -> New Terminal
```

Choose where you want to save the project, for example Desktop:

```bash
cd Desktop
```

Then download the project:

```bash
git clone https://github.com/zhangshijia992-bot/Link-up.git
```

Go into the project folder:

```bash
cd Link-up
```

## 3. Run the Website

In the VS Code terminal, type:

```bash
python run_linkup_ai_matching.py
```

If it works, the terminal will show a local website address similar to:

```text
http://127.0.0.1:4173
```

Open this link in Chrome or Edge:

```text
http://127.0.0.1:4173
```

Do not close the terminal while using the website. The Python terminal is the local server. If you close it, the website will stop working.

## 4. Alternative Way to Start

You can also double-click this file:

```text
Start Link-up AI matching.bat
```

Then open:

```text
http://127.0.0.1:4173
```

If double-clicking does not work, use the terminal method in Section 3.

## 5. Important: Gemini API Key

The Gemini API key is **not uploaded to GitHub**.

This is intentional because API keys are private. If we upload the key, other people could use it, and it may cause security or quota problems.

So after your teammates pull the project, they can still run the website, but they may see:

```text
Fallback AI
```

This means Gemini is not connected on their computer. The website still works because it has a local fallback matching engine.

### Option A: Use Without Gemini

If you only want to test the website, you can do nothing.

The platform will still:

- show AI Matching results
- calculate compatibility scores
- display explainable matching reasons
- allow Send Request
- allow Manual Search, Team Creation, Competitions, Messages, Lifecycle, and Profile pages

The only difference is that the result is generated by the local fallback AI logic instead of the Gemini API.

### Option B: Connect Gemini

If you want Gemini to generate smarter explanations:

1. Run the website.
2. Open:

```text
http://127.0.0.1:4173
```

3. Go to **AI Matching**.
4. Click **Configure Gemini Key**.
5. Paste your own Gemini API key.
6. Save it.
7. Run AI Matching again.

The key will be saved locally on your own computer as:

```text
gemini_api_key.txt
```

This file is ignored by GitHub and will not be uploaded.

## 6. How to Use the Platform

### Home

The Home page introduces the Link-Up 2.0 prototype and shows the main functions:

- AI Matching
- Manual Search
- Team Creation
- Competitions

You can also click **Watch Link-Up Intro** to view the visual and audio introduction.

### AI Matching

This is the main coded function.

There are three modes:

1. **Find Individual Collaborator**
   - Use this when you want to find one suitable teammate.
   - Select the project type, preferred teammate role, required skills, interests, and priority.
   - Click **Apply Matching Preferences** first.
   - Then click **Run AI Matching**.

2. **Find a Team to Join**
   - Use this when you want to join an existing team.
   - Select the type of team you want, your target role, preferred team goal, and team requirements.
   - Click **Apply Matching Preferences**.
   - Then click **Run AI Matching**.

3. **Recruit Teammates for My Team**
   - Use this after you already have a team.
   - Select which team you want to recruit for.
   - Select one open role at a time.
   - Set role-specific required skills.
   - Click **Apply Matching Preferences**.
   - Then click **Run AI Matching**.

The result will show:

- candidate name or team name
- compatibility score
- skills
- AI explanation
- Send Request button

### Manual Search

Manual Search is for users who do not want automatic recommendations.

You can search:

- individuals
- teams
- people to invite to your own team

The search supports different letter cases. For example:

```text
Machine Learning
machine learning
ML engineer
ml engineer
```

These can still find relevant Machine Learning profiles.

Click **View Profile** to see more details about a person.

### Team Creation

Use this page to create your own team.

You can enter:

- team name
- project type
- team size
- required roles
- team description

After publishing, the team will be saved into the CSV storage. When you come back later, the team should still exist.

### Competitions

This page shows innovation challenges and competitions.

Each competition includes:

- introduction
- requirements
- reward
- existing teams

Users can apply to join an existing team or create a team for that competition.

### Messages

Messages show chats that are unlocked after collaboration is accepted or after a team is created.

The platform does not allow random private messages between unrelated users. This supports privacy, professionalism, and focused collaboration.

### Lifecycle

Lifecycle tracks collaboration status.

Example statuses:

- Pending
- Active
- Completion Requested
- Completed
- Leave Requested
- Ended

This helps users manage collaboration after matching, not only before matching.

### Profile

Profile shows user information such as:

- role
- skills
- interests
- availability
- portfolio
- badges

The profile data is connected to the CSV prototype storage.

### Voice Assistant

There is a small circular voice assistant button.

You can drag it to another position.

Click it once to play a voice summary of the current page.

Click it again while it is speaking to stop the voice.

If you click again after stopping, it starts from the beginning.

## 8. Why CSV Is Used

CSV is used because this is a high-fidelity alpha prototype. It is simple, easy to inspect, and suitable for demonstrating the AI Matching function.

In a real production version, CSV should be replaced by a cloud database such as:

- Firebase
- MongoDB Atlas
- MySQL
- PostgreSQL

The cloud database would support more users, better security, real-time updates, and stronger data management.

## 9. Common Problems

### Problem: The website does not open

Make sure the terminal is still running:

```bash
python run_linkup_ai_matching.py
```

Then open:

```text
http://127.0.0.1:4173
```

### Problem: Python command does not work

Try:

```bash
py run_linkup_ai_matching.py
```

If that works, use `py` instead of `python`.

### Problem: It says Fallback AI

This is not a bug.

It means Gemini API is not configured or Gemini is temporarily unavailable. The prototype still works using the local fallback matching engine.

### Problem: Gemini is slow or unavailable

Sometimes Gemini returns high-demand or timeout errors. This depends on Google servers and free quota.

The platform will automatically use fallback AI so the demo can continue.

### Problem: Changes do not appear in browser

Press:

```text
Ctrl + F5
```

This forces the browser to refresh the latest CSS and JavaScript.
